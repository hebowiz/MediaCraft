# MediaCraft 仕様書

## 1. 概要

広告を含まず、一般的な動画再生機能に加えて、フレーム単位での映像確認を快適に行えるWindows向けデスクトップ動画プレイヤーを開発する。

多様な動画・音声Codecへの対応は既存の再生エンジンへ任せ、アプリケーション側ではUI、プレイリスト、フレーム操作、スクリーンショット、A-Bリピートなどの機能を実装する。

本仕様書は初期開発用であり、将来的な機能追加を前提とする。

---

## 2. 開発方針

### 2.1 対象環境

* OS：Windows 10 / Windows 11
* 言語：Python 3.11.9を基準とする
* GUI：PySide6
* 再生バックエンド：通常形式はmpv / libmpv、AMV4 AVIはWindows Media Player ActiveX経由のDirectShow/VfW
* メディア情報取得：mpvプロパティ、フレームレート判定にはPyAV。ただしAMV4 AVIは安全なRIFFヘッダー解析を使用する
* 設定保存：QSettings
* 配布形式：将来的にPyInstallerによるexe化を想定

### 2.2 推奨構成

```text
PySide6 GUI
    ↓
PlayerController
    ↓
PlayerBackend 抽象インターフェース
    ├─ 通常形式 → libmpv → FFmpeg系デコーダ
    └─ AMV4 AVI → Windows Media Player → DirectShow/VfW → インストール済みAMV4コーデック
```

動画のデコード処理は自作せず、mpv/libmpvへ委譲する。

UIと再生バックエンドを直接密結合させず、`PlayerController`を介して操作する。

### 2.3 バックエンド選定

基本構成は以下を推奨する。

* PySide6
* python-mpvまたはlibmpvのPythonバインディング
* Qtウィジェット内への映像描画

フルスクリーン時に半透明UIを映像へ重ねる必要があるため、可能であればQtの描画領域へlibmpvの映像を描画する構成とする。

mpvを別プロセスとして起動するJSON IPC方式を採用する場合も、将来的にlibmpv方式へ置き換えられるよう、バックエンドを抽象化する。

### 2.4 初期実装の決定事項

初期実装では、以下を採用する。

* アプリケーション名は`MediaCraft`とする
* Python 3.11.9で動作する構成とし、Python 3.12以降に固有の機能は使用しない
* 依存ライブラリはPython 3.11対応版を選定し、バージョンを依存定義ファイルで管理する
* 再生バックエンドは`python-mpv`を介してlibmpvを利用し、Qtウィジェット内へ映像を描画する
* AMV4 AVIはFourCCをデコーダーを起動せずに判定し、Windows Media Player ActiveX経由でインストール済みの64bit版AMV4 VfWコーデックを利用する
* AMV4 AVIをPyAVの同一プロセス解析へ渡さず、未対応コーデックによるネイティブクラッシュからアプリ本体を保護する
* AMV4コーデックはMediaCraftへ同梱せず、利用者が正規にインストールしたWindowsコーデックを使用する
* AMV4の再生速度変更はフレームクロック再生で実現し、1.00倍以外では音声なしとする
* 開発時はローカル環境のmpv/libmpvを参照し、配布対応時に必要なDLLの同梱方法を整備する
* 設定保存にはQSettingsを使用する
* UIはWindowsで見やすいダークテーマを基本とし、標準的で理解しやすい操作性を優先する
* 停止時は再生を停止し、再生位置を動画の先頭へ戻す
* 動画表示領域のシングルクリックによる再生／一時停止は初期状態で有効とし、将来的に設定で無効化できる構成とする
* フルスクリーン時は、マウス操作で表示される半透明オーバーレイUIを使用する

Python 3.11.9では実現できない要件が判明した場合は、Pythonの更新や代替実装を行う前に理由と影響を確認する。

## 3. 目的

本アプリケーションでは、以下を実現する。

* 広告のないローカル動画再生
* 多様な動画・音声Codecへの対応
* 一般的な動画プレイヤーとしての操作性
* フレーム単位での送り・戻し
* 動画内容の確認や解析に使いやすい表示
* 複数動画の連続再生
* フルスクリーン時の没入感と操作性の両立
* 将来的な機能追加が容易な構成

## 5. 機能要件

## 5.1 ファイル読み込み

### 5.1.1 ファイル選択

メニューまたはボタンから、動画ファイルを選択して再生できること。

複数ファイルの同時選択に対応する。

### 5.1.2 ドラッグ＆ドロップ

以下のドロップ操作に対応する。

* 動画ファイル1件のドロップ
* 動画ファイル複数件のドロップ
* 動画を含むフォルダのドロップ

複数ファイルまたはフォルダがドロップされた場合は、対象動画をプレイリストへ追加する。

### 5.1.3 フォルダ内ファイル

フォルダがドロップされた場合、初期仕様では直下の対応動画ファイルを対象とする。

サブフォルダを再帰検索するかどうかは、将来的に設定項目として追加可能な構成とする。

### 5.1.4 対応拡張子

拡張子による厳密な制限はなるべく設けず、mpvが再生可能なファイルを受け付ける。

例：

* MP4
* MKV
* AVI
* MOV
* WMV
* WebM
* MPEG
* MTS
* M2TS
* TS
* FLV
* その他mpvが対応する形式

再生できないファイルはエラー表示し、アプリケーション全体は終了させない。

---

## 5.2 基本再生

以下の基本操作に対応する。

* 再生
* 一時停止
* 停止
* 次のファイル
* 前のファイル
* 指定位置へのシーク
* 数秒単位の送り
* 数秒単位の戻し
* 動画終了時の次ファイル自動再生

停止操作では再生位置を動画の先頭へ戻す。

再生終了時の動作は、リピートボタンで以下の順に切り替える。

* OFF：次のファイルを再生する。プレイリスト末尾では先頭ファイルを選択し、先頭位置で停止する
* ON：プレイリスト末尾から先頭へ移動して再生を継続する
* 1ファイルリピート：現在のファイルを先頭から再生する

既定値はOFFとする。1ファイルリピートはファイルの自動終了時だけに適用し、手動の前／次ファイル操作は妨げない。

再生速度変更時は、可能な限り音程を維持する。

### 5.3.4 速度リセット

ワンアクションで`1.00x`へ戻せること。

## 5.4 フレーム単位操作

### 5.4.1 通常モード

一時停止中に以下を実行できること。

* 1フレーム送り
* 1フレーム戻し

再生中にフレーム操作を実行した場合は、自動的に一時停止してから移動する。

### 5.4.2 フレーム確認モード

フレーム単位の確認に特化した専用モードを設ける。

仮称：

```text
フレーム確認モード
Frame Inspection Mode
```

このモードでは以下を実現する。

* 左右キー長押しによる連続フレーム移動
* キーリピート中も可能な限り滑らかに更新
* 1フレーム単位の送り・戻し
* 複数フレーム単位の送り・戻し
* 現在フレーム番号の強調表示
* 現在時刻の高精度表示
* 再生コントロールUIの簡略化
* フレーム操作中の音声ミュート
* フレーム移動終了後も一時停止を維持

推奨操作：

```text
← / →            1フレーム戻し / 送り
Shift + ← / →    10フレーム戻し / 送り
Ctrl + ← / →     100フレーム戻し / 送り
長押し            連続フレーム移動
```

### 5.4.3 フレーム戻しの実装上の注意

圧縮動画では、前フレームへ戻る際に直前のキーフレームから再デコードが必要になる場合がある。

そのため、以下を許容する。

* フレーム送りよりフレーム戻しが遅い場合がある
* Codecや動画構造によって滑らかさが異なる
* 長いGOPを持つ動画では応答が遅くなる場合がある

フレーム戻し操作が連続入力された場合は、すべての操作を無制限にキューへ蓄積しない。

最新の目標位置を優先し、古い操作要求を適宜破棄または統合する。

### 5.4.4 高速フレーム操作

連続フレーム操作では、以下の方式を検討する。

1. mpvの`frame-step`および`frame-back-step`を使用する
2. 一定速度の超低速再生を利用する
3. フレームキャッシュを利用する
4. 近傍フレームを事前デコードする
5. 必要に応じてPyAVまたはFFmpegによる専用フレーム解析機能を追加する

初期実装ではmpvのフレーム操作機能を使用する。

操作性に問題がある場合は、フレーム確認モード専用のキャッシュまたはフレームインデックスを追加する。

## 5.5 フレーム番号表示

現在表示中のフレーム番号を表示する。

表示例：

```text
Frame: 12,345 | 29.97 FPS
```

### 5.5.1 CFR動画

固定フレームレート動画では、以下を基準にフレーム番号を算出できる。

```text
frame_number ≒ current_time × fps
```

必要に応じてmpvが提供する推定フレーム番号を使用する。

### 5.5.2 VFR動画

可変フレームレート動画では、単純な`時刻 × FPS`では正確なフレーム番号にならない。

初期実装では推定フレーム番号として表示し、フレームレート欄を`VFR`とすることで参考値であることを示す。

表示例：

```text
Frame: 12,345 | VFR
```

将来的にはFFprobeまたはPyAVでフレームPTSのインデックスを作成し、より正確なフレーム番号表示へ対応可能とする。

### 5.5.3 表示開始番号

初期仕様では先頭フレームを`0`とする。

将来的に設定で`0始まり`と`1始まり`を切り替えられる構成が望ましい。

## 5.6 シークバー

### 5.6.1 基本機能

シークバーには以下を実装する。

* 現在位置の表示
* クリック位置への移動
* ドラッグによる連続シーク
* 読み込み済み位置の表示
* A-Bリピート区間の表示
* 現在位置の時刻表示

### 5.6.2 シーク中の動作

シークバーのつまみをドラッグしている間は、一定間隔でプレビュー位置を更新する。

更新頻度を高くしすぎて再生処理を詰まらせないよう、スロットリングを行う。

推奨更新間隔：

```text
30～100 ms
```

シーク操作終了時には、最終位置へ正確に移動する。

### 5.6.3 サムネイルプレビュー

シークバーへマウスカーソルを合わせた際、対象位置のサムネイルを表示する。

表示内容：

* 対象位置の画像
* 対象時刻
* 必要に応じてフレーム番号

表示例：

```text
┌─────────────────┐
│  Thumbnail      │
│  00:12:34.567   │
│  Frame 18,519   │
└─────────────────┘
```

### 5.6.4 サムネイル生成方式

以下の方式を検討する。

* mpvによる対象位置の画像取得
* FFmpegによるサムネイル生成
* 動画読み込み時の事前生成
* 初回参照時の遅延生成
* メモリまたは一時ファイルへのキャッシュ

初期実装では、サムネイル機能を独立モジュールとして実装する。

MediaCraftの初期実装では、PyAVでサムネイルを遅延生成する。ホバー位置は
50 msの待機後に生成要求し、2秒間隔の近傍位置では同じ画像を再利用する。
生成処理はGUIスレッド外で実行し、動画切り替え前の古い生成結果は表示しない。
古い要求のデコード中も最新位置を待たせにくくするため、生成は最大2並列とする。
アプリ終了時は生成中のタスクをキャンセルし、サムネイル完了待ちで終了を妨げない。

動画の総時間取得後は、低解像度の粗いサムネイルをバックグラウンド生成する。
最初に動画全体を最大40区間で覆い、その後、最低5秒間隔かつ最大240枚まで
段階的に補完する。ホバー時は利用可能な近傍画像を即座に表示し、指定位置の
サムネイル生成後に高解像度画像へ差し替える。先読みは低優先度で実行し、
ホバー位置の生成と動画再生を優先する。

粗い先読みでは、指定位置以前の最寄りのキーフレームへ移動し、最初に復号できた
フレームを即座に採用する。指定時刻までの追加デコードは行わず、速度を優先する。
表示時刻と正確に一致する画像は、ホバー要求による高解像度生成で補う。

推奨クラス：

```text
ThumbnailProvider
ThumbnailCache
```

長時間動画で大量のサムネイルを生成しないよう、以下を考慮する。

* 一定時間間隔で生成
* 近い位置では同一サムネイルを再利用
* 最大キャッシュ数を設定
* LRU方式で古いサムネイルを破棄
* 動画ファイル変更時に不要キャッシュを破棄

初期実装のメモリキャッシュ上限は48枚とし、LRU方式で破棄する。

サムネイル機能は初期MVPの後に追加してもよい。

## 5.8 スクリーンショット

現在表示している映像フレームを画像として保存できること。

### 5.8.1 保存対象

以下を選択できる構成とする。

* 動画映像のみ
* 字幕を含む映像
* 将来的にプレイヤーUIを含む画面全体

初期実装では「動画映像のみ」を既定値とする。

### 5.8.2 保存形式

最低限、以下に対応する。

* PNG
* JPEG

既定値：

```text
PNG
```

### 5.8.3 ファイル名

既定のファイル名には以下を含める。

* 元動画ファイル名
* フレーム番号
* 保存日時（秒単位）

例：

```text
sample_frame012345_20260720_123456.png
```

### 5.8.4 保存先

以下から選択できる。

* 設定済みのスクリーンショットフォルダ
* 元動画と同じフォルダ
* 保存時に毎回選択

初期実装では、設定済みフォルダへ保存する。

MediaCraftの初期実装では以下とする。

* 既定の保存先は、ユーザーの「ピクチャ」フォルダ内の`MediaCraft`フォルダ
* 保存先は「ファイル」メニューから変更可能
* 保存形式はPNGまたはJPEGを「ファイル」メニューから選択可能
* `Ctrl + S`で現在フレームを保存
* 保存対象は動画映像のみとし、字幕・OSD・MediaCraftのUIは含めない
* ファイル名には元動画名、フレーム番号、保存日時（秒単位）を含める

保存成功時は、画面上に短時間の通知を表示する。

例：

```text
スクリーンショットを保存しました
```

通知は再生を妨げないトースト形式とする。

## 5.10 A-Bリピート

動画内の指定区間を繰り返し再生できること。

### 5.10.1 基本操作

以下を実装する。

* A点設定
* B点設定
* A-Bリピート開始
* A-Bリピート解除
* A点のみ解除
* B点のみ解除
* A点への移動
* B点への移動

キーボードショートカットは以下とする。

* `A` / `B`：現在位置をA点／B点に設定
* `R`：A-Bリピート切り替え
* `Shift+A` / `Shift+B`：A点／B点を解除

A点／B点への移動は再生メニューから操作し、専用ショートカットは割り当てない。

### 5.10.2 動作条件

* A点とB点の両方が設定された時点でリピートを有効化できる
* `A < B`であること
* B点到達時にA点へ戻る
* 有効化時の現在位置がA-B区間外の場合はA点へ移動する
* 有効化時の再生／一時停止状態は維持する
* 有効中に手動操作でA-B区間外へ移動した場合はリピートのみ解除し、A点とB点は保持する
* 再生によるB点到達は手動操作とは扱わず、A点へ戻ってリピートを継続する
* A点またはB点は再設定できる
* ファイル変更時は原則としてA-B区間を解除する

### 5.10.3 UI表示

シークバー上にA点とB点を表示する。

A-B区間は、通常区間と見分けられる表示にする。

表示例：

```text
A: 00:01:20.000
B: 00:01:35.500
```

### 5.10.4 フレーム単位設定

フレーム確認モードでは、現在フレームをA点またはB点として設定できる。

## 5.11 プレイリスト

### 5.11.1 表示項目

プレイリストには最低限、以下を表示する。

* ファイル名
* 再生時間
* 現在再生中であることを示す表示

将来的に以下を追加可能とする。

* ファイルサイズ
* 再生済み状態
* サムネイル

### 5.11.2 操作

以下を実装する。

* ファイル追加
* 複数ファイル追加
* フォルダ追加
* 選択ファイル削除
* 全件削除
* 並び替え
* ドラッグによる順序変更
* ダブルクリックで再生
* 次のファイル
* 前のファイル

停止中または一時停止中に、現在の再生項目とは異なるプレイリスト項目を選択して再生操作を行った場合は、選択項目を読み込んで再生する。再生中の再生／一時停止操作は、プレイリスト上の選択状態にかかわらず現在の動画を一時停止する。
* シャッフル再生ON／OFF
* リピート再生OFF／ON／1ファイルリピート

再生中の項目を削除した場合は、残った項目を選択して先頭位置で停止する。プレイリストが空になった場合は、読み込み中の動画を閉じて初期画面へ戻す。

シャッフル再生では、現在のファイルを除く未再生項目をランダムな順序で一巡する。同じファイルの連続再生は避け、前ファイル操作ではシャッフル履歴を戻る。プレイリスト全体のリピートと併用した場合は、一巡後に再度順序をシャッフルする。

シャッフルとリピートは、一般的な交差矢印と循環矢印のアイコンボタンで操作する。1ファイルリピート時は循環矢印内に「1」を表示し、有効状態はボタン背景でも判別できるようにする。

### 5.11.3 プレイリスト表示

* 左側サイドパネル

## 6. キーボードショートカット

すべてのキーボードショートカットはアプリケーション全体を対象とし、通常表示、フルスクリーン表示、フルスクリーンのオーバーレイUI表示中／非表示中のいずれでも有効であること。

例外として、プレイリスト表示切り替えはフルスクリーン中にプレイリストを表示しない設計と競合するため無効化する。通常表示へ戻った際は、フルスクリーン移行前のプレイリスト表示状態を復元し、表示切り替え操作も再度有効化する。

今後ショートカットを追加する場合も、非表示になるメニューやコントロール固有のショートカットへ直接依存させず、アプリケーション共通のショートカット登録処理を使用する。

| →                | 5秒送り          |

| Shift + →        | 30秒送り         |

| .                | 1フレーム送り       |
| F                | フルスクリーン切替     |
| Esc              | フルスクリーン解除     |
| M                | ミュート切替        |
| ↑                | 音量を上げる        |
| ↓                | 音量を下げる        |

## 15. 性能要件

* UI操作に対して即座に視覚的フィードバックを返す
* 通常操作でGUIをフリーズさせない
* シークバー操作中にイベントを過剰送信しない
* フレーム連続操作時にコマンドを無制限に蓄積しない
* 長時間動画でもメモリ使用量が増え続けない
* サムネイルキャッシュに上限を設ける
* プレイリストへ多数のファイルを追加してもUIを長時間停止させない
* 4K動画ではハードウェアデコードを利用可能にする
* フルスクリーンUIの表示・非表示が再生を妨げない

PyAV、python-mpv、PyInstaller等のライセンス全文を同梱する。依存関係または
libmpvビルドを更新した場合は、配布前にライセンスと対応ソースの案内を再確認する。

## 18. 初期MVPの完成条件

状態：完了（2026-07-20）

以下がすべて動作した時点を、初期MVP完成とする。

* 動画ファイルを選択して再生できる
* 動画をドラッグ＆ドロップして再生できる
* 再生／一時停止できる
* シークできる
* 現在時刻と総時間が表示される
* 音量変更とミュートができる
* 再生速度を変更できる
* 1フレーム送り・戻しができる
* フレーム番号が表示される
* 複数ファイルをプレイリスト再生できる
* A-Bリピートができる
* スクリーンショットを保存できる
* フルスクリーンに切り替えられる
* フルスクリーン時に半透明UIを表示できる
* 無操作時にUIを非表示にできる
* 動画の再生エラーでアプリ全体が終了しない

シークバーのサムネイル表示は、初期MVP後の追加機能としてもよい。

## 19. 実装上の制約

* UIからmpvを直接操作せず、ControllerまたはBackendを経由する
* GUIスレッドで重い処理を実行しない
* 絶対パスをソースコードへ固定しない
* OS固有処理は可能な限り分離する
* 例外を握り潰さない
* `print()`ではなく`logging`を使用する
* 1つの巨大な`main.py`へすべてを実装しない
* MVP段階で過剰な抽象化を行わない
* フレーム番号を常に厳密値として扱わない
* VFR動画では推定フレーム番号となる可能性を考慮する
* mpvおよびFFmpegのライセンス条件を配布前に確認する
* `LICENSE`、`THIRD_PARTY_NOTICES.md`、`LICENSES`を配布物から削除しない

## 20. Codexへの実装指示

以下の方針で実装すること。

1. 最初から全機能を一括実装しない
2. Phase 1から順番に実装する
3. 各Phaseの完了時に動作確認を行う
4. 既存機能を壊さないよう、小さな差分で変更する
5. 再生バックエンドとUIを分離する
6. フレーム操作は通常シークと分離して管理する
7. UIをブロックする処理は非同期化する
8. 新しい依存ライブラリを追加する場合は理由を説明する
9. 実装後に起動手順と確認手順をREADMEへ記載する
10. 不明点があっても、まずは妥当な初期値でMVPを構築する
11. フレーム番号の厳密性について、CFRとVFRを区別する
12. サムネイル機能は再生機能から独立したモジュールとする
13. フルスクリーンUIは動画描画領域より手前に表示する
14. アプリ終了時にスレッド、mpv、タイマーを正常終了させる
